TICKET #003: NORMIE MODE MEMORY ISOLATION
================================================================================
Status: 🔴 OPEN
Priority: P1 - HIGH
Severity: MEDIUM
Reported: 2026-04-20 20:21
Component: pi_agent.py - mode switching, memory access

DESCRIPTION:
When switching from root mode to normie mode, Pi loses all context from
the root mode session. This creates jarring discontinuity mid-conversation.

EVIDENCE:
```
[In root mode]
Ash: remember these 5 things... [stored successfully]

Ash: normie mode
[Switches to normie]

Ash: what things did i tell u to remember?

Pi: You told me to remember that you just switched to NORMIE MODE.

Ash: before that what did i tell u to remember?

Pi: Before switching to NORMIE MODE, I don't have any specific memories 
of you telling me to remember anything.
```

ROOT CAUSE:
Normie mode (Groq) doesn't have access to:
1. Memory tools (expected - normie has no tools)
2. Conversation history from root mode
3. L3 context loaded at startup

ANALYSIS:
This might be intentional (normie = stateless for cost savings), but creates
terrible UX. User doesn't understand why Pi suddenly "forgets" everything.

DESIGN DECISION NEEDED:
Should normie mode have:
- Read-only memory access? (can see L3, can't write)
- Full conversation history from current session?
- Or stay completely stateless?

REPRODUCTION STEPS:
1. Start in root mode
2. Have conversation, store items
3. Switch to normie mode
4. Ask about previous conversation
5. OBSERVE: Normie has no memory of it

EXPECTED BEHAVIOR (Option A):
Normie mode has read-only access to L3 context + session history

EXPECTED BEHAVIOR (Option B):
Normie mode clearly states its limitations upfront

ACTUAL BEHAVIOR:
Normie mode acts like conversation just started, confusing user

IMPACT:
- Confusing UX
- User doesn't know what Pi "knows"
- Breaks conversation flow

FILES AFFECTED:
- pi_agent.py: _respond_normie() method
- prompts/consciousness.txt: normie mode explanation

DEPENDENCIES:
None

ASSIGNED TO:
VSCode Claude + Ash (design decision needed)

FIX PRIORITY:
P1 - Fix after critical bugs

VERIFICATION STEPS:
[After design decision made]
1. Root mode conversation
2. Switch to normie
3. Ask about previous context
4. Should either:
   A) Recall it (if read-only memory enabled), OR
   B) Explain limitations clearly
